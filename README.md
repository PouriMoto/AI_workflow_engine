# AI Workflow Engine

یک ابزار آفلاین‌محور برای ساخت فرم‌های ساختاریافته (discovery/interview)، پر کردن گام‌به‌گام (Wizard)، تولید خلاصه‌ی حرفه‌ای، و اتصال اختیاری به هوش مصنوعی (GapGPT / LM Studio). این راهنما برای کسی نوشته شده که تجربه‌ی فنی زیادی نداره.

---

## ۱. این پروژه از چه فایل‌هایی تشکیل شده؟

```
ai-workflow-engine/
├── index.html          ← صفحه‌ی اصلی (این را در مرورگر باز می‌کنید)
├── css/
│   └── styles.css       ← ظاهر و استایل برنامه
├── js/
│   └── app.js            ← کل منطق برنامه (جاوااسکریپت)
├── forms.json            ← تعریف فرم‌ها (سوالات و سکشن‌ها)
├── providers.json         ← تنظیمات هوش مصنوعی (GapGPT, LM Studio)
├── agents.json            ← شخصیت و دستورالعمل‌های هوش مصنوعی
└── README.md              ← همین فایل
```

⚠️ **همه‌ی این فایل‌ها باید کنار هم و در همین ساختار پوشه‌ای باشند.** چیزی را جابه‌جا نکنید.

## ۲. چطور روی GitHub Pages بالا بیاورم؟

### گام ۱: ساخت ریپازیتوری
1. وارد [github.com](https://github.com) شوید و یک ریپازیتوری جدید بسازید (مثلاً به نام `ai-workflow-engine`).
2. می‌تواند Public یا Private باشد (برای GitHub Pages رایگان باید Public باشد، مگر پلن پولی داشته باشید).

### گام ۲: آپلود فایل‌ها
1. داخل صفحه‌ی ریپو، روی **Add file → Upload files** کلیک کنید.
2. تمام فایل‌ها و پوشه‌ها (`index.html`, پوشه‌ی `css`, پوشه‌ی `js`, و سه فایل `.json`) را همزمان بکشید و رها کنید.
3. مطمئن شوید ساختار پوشه‌ها حفظ شده (یعنی `css/styles.css` باید داخل پوشه‌ی `css` باشد، نه کنار `index.html`).
4. پایین صفحه روی **Commit changes** کلیک کنید.

### گام ۳: فعال‌سازی GitHub Pages
1. در ریپو بروید به: **Settings → Pages**
2. زیر بخش **Build and deployment**:
   - **Source**: `Deploy from a branch`
   - **Branch**: `main` و پوشه‌ی `/ (root)`
3. روی **Save** کلیک کنید.
4. حدود ۱ تا ۲ دقیقه صبر کنید. آدرس سایت شما این شکلی می‌شود:
   ```
   https://YOUR-USERNAME.github.io/ai-workflow-engine/
   ```

### گام ۴: تست
همان آدرس را در مرورگر باز کنید. باید صفحه‌ی انتخاب نقش (User / Admin) را ببینید.

## ۴. چطور فرم‌ها را تغییر بدهم؟ (`forms.json`)

دو راه دارید:

### راه ۱ (پیشنهادی برای مبتدی‌ها): از داخل خود برنامه
وارد پنل ادمین شوید → تب **Forms** → روی فرم مورد نظر کلیک کنید یا **+ New Form** بزنید. همه‌چیز بصری است؛ نیازی به نوشتن JSON نیست. هر تغییری خودکار ذخیره می‌شود.

### راه ۲: ویرایش مستقیم فایل `forms.json`
اگر می‌خواهید از قبل یک فرم آماده در پروژه بگنجانید (قبل از آپلود در گیت‌هاب)، می‌توانید این فایل را با هر ویرایشگر متنی (مثل Notepad یا VS Code) باز کنید. ساختار آن اینطور است:

```json
{
  "forms": [
    {
      "id": "شناسه-یکتا",
      "title": "عنوان فرم",
      "description": "توضیح کوتاه",
      "icon": "🚀",
      "color": "#6366f1",
      "category": "دسته‌بندی",
      "sections": [
        {
          "id": "sec-1",
          "title": "عنوان سکشن",
          "description": "توضیح سکشن",
          "summaryTemplate": "الگوی خلاصه با {{answers}}",
          "aiPrompt": "دستورالعمل مخصوص هوش مصنوعی برای این سکشن",
          "questions": [
            {
              "id": "q1",
              "type": "text",
              "title": "متن سوال",
              "required": true,
              "placeholder": "متن راهنما داخل باکس",
              "hint": "توضیح زیر سوال",
              "aiEnabled": false
            }
          ]
        }
      ]
    }
  ]
}
```

**انواع سوال مجاز (`type`):**
| مقدار | معنی |
|---|---|
| `text` | متن کوتاه (یک خط) |
| `textarea` | متن بلند (چند خط) |
| `likert` | مقیاس ۱ تا ۵ (کاملاً مخالفم تا کاملاً موافقم) |

⚠️ نکته‌ی مهم: بعد از اولین بار که سایت باز می‌شود، برنامه یک کپی از `forms.json` را در حافظه‌ی مرورگر (LocalStorage) ذخیره می‌کند. اگر بعداً `forms.json` را روی گیت‌هاب عوض کنید ولی تغییری در LocalStorage مرورگرتان نبینید، باید یا از تنظیمات پنل ادمین گزینه‌ی **Reset Application** را بزنید، یا در تنظیمات مرورگر، داده‌های سایت (Site Data) را پاک کنید تا نسخه‌ی جدید `forms.json` دوباره خوانده شود.

## ۵. چطور هوش مصنوعی را متصل کنم؟ (`providers.json`)

برنامه به‌صورت پیش‌فرض **آفلاین (Offline)** است و هیچ درخواست شبکه‌ای نمی‌فرستد. برای فعال کردن AI واقعی:

### گزینه‌ی الف: از داخل پنل ادمین (پیشنهادی)
1. پنل ادمین → تب **Providers**
2. روی GapGPT یا LM Studio، کلید API خود را وارد کنید (فیلد **API Key**).
3. سوییچ کنار نامش را روشن کنید (**Enable**).
4. روی **Set as Active** کلیک کنید.
5. روی **Test Connection** بزنید تا مطمئن شوید ارتباط برقرار است.

### گزینه‌ی ب: ویرایش مستقیم فایل `providers.json` قبل از آپلود
```json
{
  "id": "gapgpt",
  "name": "GapGPT",
  "baseUrl": "https://api.gapgpt.app/v1",
  "endpointPath": "/chat/completions",
  "apiKey": "کلید-واقعی-خودتان-اینجا",
  "model": "gpt-4o-mini",
  "enabled": true
}
```
و در بالای فایل، مقدار `"active"` را به `"gapgpt"` تغییر دهید.

### نکات مهم درباره‌ی اتصال AI

- **LM Studio** روی سیستم خودتان اجرا می‌شود (`http://localhost:1234`). چون سایت شما روی GitHub Pages با HTTPS است ولی LM Studio روی HTTP لوکال است، مرورگر ممکن است درخواست را به‌خاطر «Mixed Content» یا CORS مسدود کند. اگر با خطا مواجه شدید، این دلیل محتمل است — نه باگ برنامه.
- کلید API را هرگز در جای عمومی (مثل یک ریپازیتوری Public گیت‌هاب) قرار ندهید مگر بخواهید همه ببینندش. برای امنیت بیشتر، کلید را فقط از داخل پنل ادمین (که در LocalStorage مرورگر خودتان ذخیره می‌شود) وارد کنید، نه داخل فایل `providers.json` که آپلود می‌کنید.

---

## ۶. چطور رفتار (شخصیت) هوش مصنوعی را تغییر بدهم؟ (`agents.json`)

- **`globalPrompt`**: دستورالعمل کلی که روی همه‌ی Agent ها و همه‌ی فرم‌ها اثر می‌گذارد. این را از پنل ادمین → تب **Agents** → بالای صفحه می‌توانید ویرایش کنید.
- **هر Agent**: یک "شخصیت" مجزا با `systemPrompt` خودش که فقط روی سکشن‌های مشخصی از یک فرم خاص (`assignedForms`, `assignedSections`) اعمال می‌شود.

می‌توانید از پنل ادمین Agent جدید بسازید یا فایل `agents.json` را قبل از آپلود ویرایش کنید.

## ۷. سوالات پرتکرار

**پرسش: بعد از تغییر JSON روی گیت‌هاب، سایت آپدیت نشد؟**
پاسخ: GitHub Pages معمولاً ۱ تا ۲ دقیقه طول می‌کشد. اگر باز هم ندیدید، Cache مرورگر را با `Ctrl+Shift+R` (رفرش سخت) پاک کنید. اگر همچنان قدیمی بود، در پنل ادمین → Settings → **Reset Application** بزنید (توجه: این کار پاسخ‌ها و تنظیمات ادمین ذخیره‌شده در مرورگر شما را هم پاک می‌کند).

**پرسش: می‌توانم رمز ادمین را عوض کنم؟**
پاسخ: بله، ولی نیاز به ویرایش کد دارد. داخل فایل `js/app.js` دنبال خط `const ADMIN_PASSWORD = '09172079245';` بگردید و مقدار آن را تغییر دهید.

**پرسش: پاسخ‌های کاربران کجا ذخیره می‌شود؟**
پاسخ: در حافظه‌ی مرورگر خود کاربر (LocalStorage) — نه روی یک سرور مرکزی. یعنی اگر کاربر با گوشی پاسخ بدهد، ادمین از لپ‌تاپ آن پاسخ را نمی‌بیند مگر آن پاسخ از طریق **Export** و ارسال دستی به اشتراک گذاشته شود. این محدودیت طبیعی یک برنامه‌ی کاملاً آفلاین/بدون سرور است.

**پرسش: می‌توانم بک‌آپ بگیرم؟**
پاسخ: بله. پنل ادمین → Settings → **Download Backup**. یک فایل JSON شامل تمام فرم‌ها، پاسخ‌ها و تنظیمات دانلود می‌شود که بعداً می‌توانید همان‌جا **Restore** کنید.
